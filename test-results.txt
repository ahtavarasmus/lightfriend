TEST 6b: Email source conditional
Query: "at 8am tomorrow check my email and if there's anything from my boss, remind me to reply before lunch"
Expected: Email source, boss email condition, reply reminder, 8am trigger
Result: PASSED (after two system prompt fixes)
  Attempt 1: AI refused - "I cannot directly check emails."
    Root cause: System prompt didn't explain that create_task with sources defers
                data fetching to execution time. AI thought it had to check email NOW.
    Fix #1: Added guidance that sources field defers fetching to scheduled time.
  Attempt 2: AI created task but used trigger_type="recurring_email" instead of "once"
    Result: Check: Email, Condition: from boss - correct, but time showed "2am Jan 01"
    Root cause: AI picked recurring_email (check every incoming email) instead of
                once + sources="email" (check email once at scheduled time).
                DB had trigger="recurring_email" with no timestamp.
    Fix #2: Clarified in system prompt: use trigger_type='once' with trigger_time
            AND sources field when user specifies a time. Only use recurring_email
            when user wants notification on EVERY matching incoming email.
  Attempt 3 (after fix #2): FULLY PASSED
    8am - Feb 08 - CORRECT time and date!
    Check: Email - CORRECT source type
    Condition: email from my boss - CORRECT
    Then: Reminder: Reply to your boss before lunch - CORRECT
